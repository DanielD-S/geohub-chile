Fenómenos monitoreados

Qué es cada alerta, por qué importa y cuáles son los umbrales utilizados.

🔥

Incendios forestales

Detectados por satélite mediante FRP (Fire Radiative Power). Chile concentra el 99,7% de incendios por causas humanas. Las temporadas más críticas son diciembre–marzo en el eje Maule–Biobío–Araucanía.

FRP alto: > 60 MW · Botón Rojo: ignición ≥70% + viento ≥20 km/h
🌧️

Precipitaciones extremas / aluviones

En el norte árido, lluvias de baja recurrencia generan aluviones violentos. En el centro-sur, el riesgo es inundación de riberas y corte de caminos rurales en invierno.

Alerta: >30 mm/6h · Crítico: >60 mm/6h
🌡️

Heladas

Afectan cultivos (viñas, frutales, hortalizas) y a la población vulnerable. Zonas de riesgo: valles interiores, precordillera y zonas australes. Periodo crítico: mayo–agosto.

Alerta: <2°C · Crítico: <-5°C
❄️

Nevazones

Eventos urbanos en Santiago (600–900 msnm) son raros pero paralizantes. En cordillera, el déficit de nieve equivale a crisis hídrica en verano. NASA MODIS monitorea cobertura nival.

Alerta: >5 cm · Crítico: >20 cm en 24h
💨

Viento sur / temporales costeros

El viento patagónico puede superar 100 km/h en la zona austral. En la costa central genera marejadas y erosión costera. En ciudades, derriba árboles y corta la electricidad.

Alerta: >60 km/h · Crítico: >90 km/h

01
Configuración — edita solo este bloque
QGIS ▼
Antes de ejecutar el script, ajusta el diccionario CONFIG. Es el único lugar que necesitas editar. Define las rutas a tus archivos descargados y el período de análisis.

¿Dónde descargar los datos?
· comunas_chile.shp → bcn.cl/siit/mapas_vectoriales
· firms_modis_chile.csv → firms.modaps.eosdis.nasa.gov/country (requiere cuenta NASA Earthdata gratuita)

El EPSG 5361 es SIRGAS 2000 / UTM Zona 19S, el sistema oficial de Chile continental. Cambia a 5360 si trabajas en zona 18S (norte extremo).
Python

CONFIG = {
    # Carpeta raíz donde están tus datos y se guardarán los outputs
    "dir_base":       "C:/GEO_CHILE/incendios",  # Windows
    # "dir_base":     "/home/usuario/geo_chile",  # Linux / Mac

    # Archivos de entrada
    "comunas_shp":    "comunas_chile.shp",
    "regiones_shp":   "regiones_chile.shp",
    "firms_csv":      "firms_modis_chile.csv",

    # Columnas del CSV de NASA FIRMS
    "col_lat":        "latitude",
    "col_lon":        "longitude",
    "col_fecha":      "acq_date",    # formato YYYY-MM-DD
    "col_confianza":  "confidence",  # 0–100
    "col_frp":        "frp",         # Fire Radiative Power (MW)

    # Filtros
    "confianza_minima": 50,           # descarta puntos poco confiables
    "fecha_inicio":   "2023-11-01",
    "fecha_fin":      "2024-03-31",  # temporada de incendios

    # Proyección oficial Chile continental
    "epsg":           "EPSG:5361",

    # Archivos de salida
    "out_comunas_csv":  "indicadores_comunas.csv",
    "out_regiones_csv": "indicadores_regiones.csv",
    "out_shp":          "comunas_indicadores.shp",
}
02
Cargar y reproyectar capas base
PyQGIS ▼
Carga los shapefiles de comunas y regiones desde el disco y los reproyecta a UTM zona 19S. Esto es crítico: sin reproyección, los cálculos de área en grados decimales son incorrectos (los grados no tienen la misma distancia en metros en distintas latitudes).

El parámetro "OUTPUT": "memory:" guarda la capa reproyectada en RAM, sin escribir archivos temporales al disco.
Python 
from qgis.core import QgsVectorLayer, QgsCoordinateReferenceSystem
import processing

def cargar_capas():
    # 1. Leer shapefiles desde disco
    comunas  = QgsVectorLayer(ruta(CONFIG["comunas_shp"]),  "comunas",  "ogr")
    regiones = QgsVectorLayer(ruta(CONFIG["regiones_shp"]), "regiones", "ogr")

    if not comunas.isValid():
        raise Exception(f"No se pudo cargar: {ruta(CONFIG['comunas_shp'])}")

    # 2. Reproyectar a SIRGAS 2000 / UTM Zona 19S (EPSG:5361)
    #    Necesario para calcular áreas en km² correctamente
    epsg = CONFIG["epsg"]

    comunas_utm = processing.run("native:reprojectlayer", {
        "INPUT":      comunas,
        "TARGET_CRS": QgsCoordinateReferenceSystem(epsg),
        "OUTPUT":     "memory:"   # en RAM, sin archivo temporal
    })["OUTPUT"]

    regiones_utm = processing.run("native:reprojectlayer", {
        "INPUT":      regiones,
        "TARGET_CRS": QgsCoordinateReferenceSystem(epsg),
        "OUTPUT":     "memory:"
    })["OUTPUT"]

    log(f"Comunas: {comunas_utm.featureCount()} · Regiones: {regiones_utm.featureCount()}")
    return comunas_utm, regiones_utm
03
Leer y filtrar puntos de calor NASA FIRMS
CSV → Vectorial ▼
El CSV de NASA FIRMS contiene miles de filas — una por cada punto de calor detectado por satélite. Cada fila es una detección de temperatura anómala, con su coordenada, fecha y nivel de confianza.

Este paso filtra por período de análisis y confianza mínima (descartar falsas alarmas), luego construye una capa vectorial de puntos en memoria y la reproyecta a UTM para el join espacial.
Python 
import csv, datetime
from qgis.core import QgsVectorLayer, QgsFeature, QgsGeometry, QgsPointXY, QgsField
from PyQt5.QtCore import QVariant

def cargar_firms():
    # Crear capa de puntos vacía en memoria (WGS 84)
    capa = QgsVectorLayer("Point?crs=EPSG:4326", "firms", "memory")
    prov = capa.dataProvider()
    prov.addAttributes([
        QgsField("fecha",     QVariant.String),
        QgsField("confianza", QVariant.Int),
        QgsField("frp",       QVariant.Double),
    ])
    capa.updateFields()

    fecha_ini = datetime.date.fromisoformat(CONFIG["fecha_inicio"])
    fecha_fin = datetime.date.fromisoformat(CONFIG["fecha_fin"])
    conf_min  = CONFIG["confianza_minima"]

    features, total, ok = [], 0, 0

    with open(ruta(CONFIG["firms_csv"]), "r", encoding="utf-8") as f:
        for row in csv.DictReader(f):
            total += 1
            try:
                lat   = float(row["latitude"])
                lon   = float(row["longitude"])
                conf  = int(row.get("confidence", 0) or 0)
                frp   = float(row.get("frp", 0) or 0)
                fecha = datetime.date.fromisoformat(row["acq_date"])
            except: continue

            # Aplicar filtros de calidad y período
            if conf < conf_min: continue
            if not (fecha_ini <= fecha <= fecha_fin): continue

            # Crear feature (punto geográfico)
            feat = QgsFeature()
            feat.setGeometry(QgsGeometry.fromPointXY(QgsPointXY(lon, lat)))
            feat.setAttributes([row["acq_date"], conf, frp])
            features.append(feat)
            ok += 1

    prov.addFeatures(features)
    capa.updateExtents()
    log(f"FIRMS: {total} filas → {ok} puntos válidos tras filtros")

    # Reproyectar a UTM para join espacial consistente
    return processing.run("native:reprojectlayer",
        {"INPUT": capa, "TARGET_CRS": QgsCoordinateReferenceSystem(CONFIG["epsg"]),
         "OUTPUT": "memory:"})["OUTPUT"]
04
Join espacial — asignar cada foco a su comuna
Geoproceso ▼
Aquí ocurre la operación geoespacial central: para cada punto de calor, determinar en qué polígono comunal cae. El QgsSpatialIndex acelera la búsqueda — en vez de comparar cada punto con las 346 comunas, primero descarta todas las que están fuera del bounding box del punto (colisión de cajas) y solo evalúa la intersección precisa para los candidatos restantes. Esto reduce el tiempo de ~346× a ~2× comparaciones por punto.
Python 
from qgis.core import QgsSpatialIndex

def join_espacial(firms_utm, comunas_utm):
    # Índice espacial sobre polígonos comunales (búsqueda rápida)
    idx          = QgsSpatialIndex(comunas_utm.getFeatures())
    comunas_dict = {f.id(): f for f in comunas_utm.getFeatures()}

    # Detectar nombres de campos (varían según versión del SHP de BCN)
    campos = [f.name().lower() for f in comunas_utm.fields()]
    campo_com = next((c for c in campos if "comuna" in c), campos[0])
    campo_reg = next((c for c in campos if "region" in c), "region")

    # Inicializar diccionario de resultados por comuna
    resultado = {}
    for feat in comunas_utm.getFeatures():
        resultado[feat.id()] = {
            "nombre_comuna": feat[campo_com],
            "nombre_region": feat[campo_reg] if campo_reg in campos else "N/D",
            "area_km2":      feat.geometry().area() / 1_000_000,  # m² → km²
            "puntos":        []
        }

    # Asignar cada punto FIRMS a su comuna
    asignados = 0
    for pt in firms_utm.getFeatures():
        geom = pt.geometry()
        # Paso 1: candidatos por bounding box (rápido)
        for fid in idx.intersects(geom.boundingBox()):
            # Paso 2: intersección precisa del polígono
            if comunas_dict[fid].geometry().contains(geom):
                resultado[fid]["puntos"].append(pt)
                asignados += 1
                break  # un punto cae en una sola comuna

    log(f"Puntos asignados: {asignados}/{firms_utm.featureCount()}")
    return resultado
05
Calcular los 6 indicadores por comuna
Análisis ▼
Con los puntos ya asignados a cada comuna, se calculan los 6 indicadores que verás en la tabla del visor.

· n_focos — total de detecciones satelitales
· frp_total_mw — energía acumulada del fuego (proxy de destrucción)
· frp_prom_mw — intensidad media por evento
· densidad_focos_x100km2 — normaliza por superficie (compara comunas pequeñas y grandes)
· mes_peak — mes del año con más actividad (útil para estacionalidad)
· dias_con_focos — duración del período de riesgo
Python 
def calcular_indicadores(resultado):
    indicadores = []

    for fid, data in resultado.items():
        puntos = data["puntos"]
        n      = len(puntos)
        area   = data["area_km2"]

        if n == 0:
            frp_total = frp_prom = densidad = 0.0
            mes_peak  = "N/A"
            dias      = 0
        else:
            frps      = [p["frp"]   for p in puntos]
            fechas    = [p["fecha"] for p in puntos]

            frp_total = round(sum(frps), 2)
            frp_prom  = round(frp_total / n, 2)
            densidad  = round((n / area) * 100, 4) if area > 0 else 0

            # Mes con más focos (1 = enero, 12 = diciembre)
            meses = {}
            for f in fechas:
                m = datetime.date.fromisoformat(f).month
                meses[m] = meses.get(m, 0) + 1
            mes_peak = max(meses, key=meses.get)

            dias = len(set(fechas))  # días únicos con al menos un foco

        indicadores.append({
            "nombre_comuna":           data["nombre_comuna"],
            "nombre_region":           data["nombre_region"],
            "area_km2":                round(area, 2),
            "n_focos":                 n,
            "frp_total_mw":            frp_total,
            "frp_prom_mw":             frp_prom,
            "densidad_focos_x100km2":  densidad,
            "mes_peak":                mes_peak,
            "dias_con_focos":          dias,
            "_fid":                    fid,
        })

    return indicadores
06
Agregar indicadores a nivel regional
Agregación ▼
Suma y promedia los indicadores comunales para obtener una visión regional. El indicador clave aquí es pct_comunas_afectadas: qué porcentaje de comunas de la región tuvo al menos un foco. Permite comparar regiones grandes con muchas comunas versus regiones pequeñas.
Python 
def agregar_por_region(ind):
    reg = {}
    for row in ind:
        r = row["nombre_region"]
        if r not in reg:
            reg[r] = {"nombre_region":r, "n_comunas":0,
                       "com_afect":0, "n_focos":0,
                       "frp_total":0.0, "area":0.0}
        d = reg[r]
        d["n_comunas"] += 1
        d["n_focos"]   += row["n_focos"]
        d["frp_total"] += row["frp_total_mw"]
        d["area"]      += row["area_km2"]
        if row["n_focos"] > 0: d["com_afect"] += 1

    for d in reg.values():
        d["densidad"]    = round(d["n_focos"]/d["area"]*100,4) if d["area"] else 0
        d["pct_afect"]   = round(d["com_afect"]/d["n_comunas"]*100,1) if d["n_comunas"] else 0

    return list(reg.values())
07
Exportar CSV + Shapefile y cargar en QGIS
Output ▼
Genera 3 archivos: dos CSV listos para subir a este visor y un Shapefile que se carga automáticamente en QGIS con los indicadores como campos nuevos. Desde QGIS puedes aplicar simbología graduada por cualquier indicador y generar mapas.

El CSV indicadores_comunas.csv es el que debes arrastrar a la pestaña 🔥 Incendios FIRMS de este visor para ver tu análisis propio.
Python 
import csv, os
from qgis.core import QgsVectorFileWriter, QgsProject

def exportar(ind_comunas, ind_regiones, comunas_utm):
    # ── CSV comunas ──
    campos = ["nombre_comuna","nombre_region","area_km2","n_focos",
              "frp_total_mw","frp_prom_mw","densidad_focos_x100km2",
              "mes_peak","dias_con_focos"]
    with open(ruta("indicadores_comunas.csv"), "w",
               newline="", encoding="utf-8-sig") as f:
        w = csv.DictWriter(f, fieldnames=campos, extrasaction="ignore")
        w.writeheader()
        w.writerows(ind_comunas)

    # ── Shapefile enriquecido ──
    # Agregar campos de indicadores al shapefile de comunas
    capa_out = comunas_utm.materialize(QgsFeatureRequest())
    capa_out.dataProvider().addAttributes([
        QgsField("n_focos",   QVariant.Int),
        QgsField("frp_tot",   QVariant.Double),
        QgsField("densidad",  QVariant.Double),
        QgsField("mes_peak",  QVariant.Int),
    ])
    capa_out.updateFields()

    idx_ind = {r["_fid"]: r for r in ind_comunas}
    capa_out.startEditing()
    for feat in capa_out.getFeatures():
        d = idx_ind.get(feat.id(), {})
        feat["n_focos"]  = d.get("n_focos", 0)
        feat["frp_tot"]  = d.get("frp_total_mw", 0.0)
        feat["densidad"] = d.get("densidad_focos_x100km2", 0.0)
        feat["mes_peak"] = int(d.get("mes_peak", 0) or 0)
        capa_out.updateFeature(feat)
    capa_out.commitChanges()

    # Guardar SHP y agregar al proyecto QGIS
    ruta_shp = ruta("comunas_indicadores.shp")
    QgsVectorFileWriter.writeAsVectorFormat(
        capa_out, ruta_shp, "utf-8",
        QgsCoordinateReferenceSystem(CONFIG["epsg"]), "ESRI Shapefile")

    QgsProject.instance().addMapLayer(
        QgsVectorLayer(ruta_shp, "Comunas — Indicadores Incendios", "ogr"))

    log("✅ Outputs generados en: " + CONFIG["dir_base"])
